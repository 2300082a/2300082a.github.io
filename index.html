How the CakeML compiler is verified and what does it mean to be verified(technical)

The CakeML compiler is a type of software known as a compiler. Compilers play a crucial role in programming by converting code written in a programing languages into machine code that computers can understand and execute. This software is essential to be correct/bug free because if the translation is incorrect, it can lead to bugs and unexpected behavior in the software written by programmers even if their code is correctly. To ensure that the CakeML compiler works correctly, it uses a technique called software verification. This process uses mathematical methods to prove that the software meets a specification. For compilers, the specification for it's correctness is called semantics preservation. This means that the program should behave the same way after it has been compiled as to some specification of the programming language. Lets me use the language of addition and multiplication over natural numbers as an example. 2 * 3 is the semantically the same as 2 + 2 + 2 while being different syntactically aka different text. In the same way, the CakeML compiler ensures that more complex programs maintain their intended behavior after being translated into machine code.
